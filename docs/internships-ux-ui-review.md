# Раздел «Стажировки» — обзор блоков и рекомендации по UX/UI

## Текущая структура карточки стажировки

1. **Шапка**: бейдж статуса (справа) + бейдж типа программы + иконка календаря + период (даты)
2. **Разделитель**
3. **Блок «Воронка»**: Заявки (0) | Целевые заявки | Прошли
4. **Блок «Результаты»**: Стажеры (20) | Сотрудники (17) | Конверсия (85%)
5. **Разделитель**
6. **Текст**: «Нанимающих подразделений: N»
7. **Разделитель**
8. **Кнопка**: «Подробнее» со стрелкой

---

## Проблемы и идеи по улучшению

### 1. Иерархия и сканируемость

- **Сейчас**: тип программы дублируется (вкладка + бейдж в карточке). Даты и тип в одной строке с иконкой — взгляд размазывается.
- **Идеи**:
  - Сделать главным один элемент: либо **период** («15.01.2025 – 20.04.2025») крупнее, тип — вторичным текстом, либо наоборот.
  - Убрать бейдж типа из карточки при открытой вкладке этого типа (оставить только даты + статус) или оставить один лаконичный подзаголовок вместо бейджа.

### 2. Воронка: логика и визуал

- **Сейчас**: три одинаковых ячейки в ряд; «Заявки» везде 0 (нет привязки к данным заявок).
- **Идеи**:
  - Подтянуть число заявок из данных (если есть `totalApplications` или список заявок по стажировке).
  - Показать воронку явно: например, стрелки между ячейками (Заявки → Целевые → Прошли) или тонкий прогресс-бар под блоком (сколько от заявок «прошло» до целевых).
  - Первую ячейку визуально отличать (например, «вход» в воронку), последнюю — как «итог этапа».

### 3. Результаты: захардкоженные числа

- **Сейчас**: 20, 17, 85% не привязаны к стажировке — вводят в заблуждение.
- **Идеи**:
  - Выводить только те метрики, для которых есть данные (стажеры / сотрудники / конверсия из API или расчёта). Если данных нет — писать «—» или «Нет данных», а не фиксированные цифры.
  - Конверсию считать от реальных данных (например, завершённые / целевые заявки) и показывать только при наличии знаменателя.

### 4. «Нанимающих подразделений»

- **Сейчас**: отдельная строка между двумя разделителями — выглядит «висячим» блоком.
- **Идеи**:
  - Объединить в одну компактную строку-футер карточки: например, «Подразделений: 5» рядом с кнопкой «Подробнее» или в одну строку с другими сводными цифрами (если появятся).
  - Либо оформить как небольшую метрику в общем стиле ячеек (иконка + число + подпись), чтобы визуально не выделяться.

### 5. Пустые слоты в сетке

- **Сейчас**: при < 4 карточках остаются пустые серые блоки.
- **Идеи**:
  - В пустой слот добавить призыв к действию: «Добавить стажировку» (кнопка или вся область кликабельна) с плюсом — так пустое место превращается в понятное действие.

### 6. Интерактивность карточки

- **Сейчас**: кликабельна по сути только кнопка «Подробнее».
- **Идеи**:
  - Сделать кликабельной всю карточку (переход на детальную страницу), кнопку оставить визуальным акцентом. Добавить `cursor-pointer` и лёгкий hover (подсветка границы / тень), чтобы было очевидно, что карточка — ссылка.

### 7. Подписи метрик (UPPERCASE)

- **Сейчас**: подписи в верхнем регистре, мелкий шрифт — хуже читаются.
- **Идеи**:
  - Перейти на обычный регистр (Sentence case): «Заявки», «Целевые заявки», «Стажеры», «Сотрудники», «Конверсия» — так быстрее считывается.

### 8. Визуальное разделение «Воронка» / «Результаты»

- **Сейчас**: два блока подряд с одинаковой структурой — можно спутать.
- **Идеи**:
  - Небольшой отступ или подзаголовок с другим весом шрифта.
  - Либо лёгкий фон только у одного из блоков (например, только у «Воронка» или только у «Результаты»), чтобы разделить «этап отбора» и «итоги».

### 9. Узкие карточки (2 колонки)

- **Сейчас**: в одной строке три ячейки — на узких экранах могут сжиматься.
- **Идеи**:
  - На малых ширинах можно показывать метрики в два ряда (2+1) или уменьшать паддинги/шрифты только для метрик, сохраняя заголовки блоков читаемыми.

### 10. Единая кнопка действия

- **Сейчас**: одна кнопка «Подробнее» — логично.
- **Идеи**:
  - Сохранить одну основную кнопку; при необходимости позже добавить вторичное действие (например, «Быстрый просмотр» во всплывашке) без перегруза карточки.

## Приоритеты внедрения

| Приоритет | Что сделать |
|-----------|-------------|
| Высокий   | Убрать захардкоженные 20/17/85%, показывать реальные данные или «—» |
| Высокий   | Подтянуть число заявок в «Воронка» из данных (если есть источник) |
| Средний   | Кликабельность всей карточки + hover |
| Средний   | Упростить шапку (одна главная строка: даты или тип) |
| Средний   | Пустой слот = «Добавить стажировку» |
| Низкий    | Подписи в обычном регистре, визуальное разделение блоков Воронка/Результаты |
| Низкий    | Компактное размещение «Нанимающих подразделений» |

Если нужно, могу предложить конкретные правки в `internships-tab.tsx` под выбранные пункты (например, только данные + клик по карточке + пустой слот).
