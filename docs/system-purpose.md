# Назначение системы «КАМПУС»

## Определение

**КАМПУС** (quant-universities-platform) — корпоративная веб-платформа для централизованного управления сотрудничеством банка с высшими учебными заведениями.

Выделена из экосистемы Skillmap как самостоятельное решение.

## Цели системы

1. **Единый справочник ВУЗов** — консолидация информации обо всех вузах-партнёрах (российских и зарубежных), их контактных лицах, филиалах и кураторах в одном месте.
2. **Управление линиями сотрудничества** — ведение и отслеживание работы по пяти направлениям взаимодействия: БКО, ЦНТР, ДРП, Экосистема, ДКМ — каждое со своим личным кабинетом.
3. **Управление договорами** — учёт договоров сотрудничества, стипендий, стажировок и кафедр банка с контролем сроков и архивированием.
4. **Управление мероприятиями** — планирование, проведение и учёт мероприятий (деловые игры, кейс-чемпионаты, лекции, конференции, мастер-классы, вебинары, защиты дипломов) по линиям сотрудничества.
5. **Управление стажировками** — полный цикл: создание программы, приём и скоринг заявок, формирование основного и резервного списков, оценка стажёров, расчёт конверсии.
6. **Учёт персонала** — ведение реестров стажёров, практикантов, целевых практикантов, именных стипендиантов и участников кейс-чемпионатов.
7. **Аналитика и отчётность** — дашборд со сводными показателями, формирование отчётов, экспорт данных в Excel.

## Целевые пользователи

- Сотрудники подразделений банка, ответственные за взаимодействие с вузами по линиям сотрудничества (БКО, ЦНТР, ДРП, Экосистема, ДКМ).
- Кураторы от филиалов банка.
- Руководители и аналитики, использующие отчётность и дашборд.

## Границы системы

### Входит в скоуп

- Справочник ВУЗов (CRUD, поиск, фильтрация)
- Личные кабинеты по 5 линиям сотрудничества
- Реестр договоров
- Реестр и лента мероприятий (калейдоскоп)
- Модуль стажировок (заявки, скоринг, оценки, воронка)
- Реестры персонала (стажёры, практиканты, стипендианты)
- Дашборд и отчётность с экспортом в Excel
- Шаблоны email-писем для коммуникации с вузами

### Не входит в скоуп (на текущем этапе)

- Серверная часть / API (данные моковые, клиентское состояние)
- Аутентификация и авторизация
- Интеграция с внешними системами (АСДД, HR-системы)
- Личный кабинет со стороны вуза / студента

## Ключевое бизнес-правило

Центральная сущность системы — **ВУЗ**. Все остальные объекты (договоры, мероприятия, персонал, данные линий сотрудничества) агрегируются вокруг карточки вуза. Стажировки связаны с вузами, но управляются как самостоятельный модуль со своим жизненным циклом.
